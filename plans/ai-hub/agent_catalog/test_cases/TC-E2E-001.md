# TC-E2E-001: Browse and inspect agent from default catalog

**Objective**: Verify the complete user journey of browsing the default
agent catalog, searching for an agent, and retrieving its full details
and artifacts.

**Preconditions**:
- RHOAI operator deployed with agent catalog plugin enabled
- Default Red Hat starter kits catalog loaded (15 agents)

**Test Steps**:
1. Verify catalog service is healthy via GET `/healthz`
2. Verify agent plugin is ready via GET `/readyz`
3. Send GET to `/api/agent_catalog/v1alpha1/agents` to list all agents
   and confirm default agents are loaded
4. Send GET to `/api/agent_catalog/v1alpha1/agents?q=langgraph` to
   search for a specific agent framework
5. Extract the agent ID from the search results
6. Send GET to `/api/agent_catalog/v1alpha1/agents/{agent_id}` to
   retrieve full agent details
7. Send GET to
   `/api/agent_catalog/v1alpha1/agents/{agent_id}/artifacts` to
   retrieve the agent's artifacts

**Expected Results**:
- `/healthz` returns HTTP 200
- `/readyz` returns HTTP 200 with agent_catalog plugin healthy
- List agents returns 15 default agents with `size` field equal to 15
- Search results contain only agents matching the keyword `langgraph`
- Agent detail response contains complete metadata: name, description,
  customProperties with framework, agentType, tags
- Artifacts response contains at least one artifact associated with
  the agent
