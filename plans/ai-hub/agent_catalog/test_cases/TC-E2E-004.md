# TC-E2E-004: Operator deployment wires agent catalog end-to-end

**Objective**: Verify the complete operator deployment journey: ConfigMap
creation, volume mounting, init container loading, plugin registration,
and agent availability via API.

**Preconditions**:

- Fresh RHOAI operator installation or upgrade with agent catalog
  feature enabled

**Test Steps**:

1. Verify `default-catalog-sources` ConfigMap exists and contains
   `agent_catalogs` section
2. Verify the catalog service deployment has the shared-data volume
   mount
3. Verify the init container is configured to load agents-catalog.yaml
4. Verify the `--catalogs-path` argument includes the agent sources
   path
5. Verify `/healthz` returns HTTP 200
6. Verify `/readyz` returns HTTP 200 with agent_catalog plugin healthy
7. Send GET to `/api/agent_catalog/v1alpha1/agents` and verify agents
   are loaded from the default catalog
8. Verify
   `/api/model_catalog/v1alpha1/sources?assetType=agents` returns
   the default agent source

**Expected Results**:

- ConfigMap contains agent_catalogs with Red Hat starter kits reference
- Deployment has correct volume mount and init container configuration
- `--catalogs-path` argument is present in container args
- Both health and readiness endpoints return HTTP 200
- 15 agents from the default catalog are accessible via the API
- Agent sources appear in the unified sources endpoint
