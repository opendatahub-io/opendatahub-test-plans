# TC-E2E-002: Add custom agent source and browse combined catalog

**Objective**: Verify the complete user journey of adding a custom agent
catalog source via ConfigMap, then browsing agents from both default and
custom sources with filtering by source.

**Preconditions**:

- RHOAI operator deployed with agent catalog plugin enabled
- Default Red Hat starter kits catalog loaded
- Custom agent catalog YAML prepared with 7 agents

**Test Steps**:

1. Verify initial agent count from default source via GET
   `/api/agent_catalog/v1alpha1/agents`
2. Patch `model-catalog-sources` ConfigMap to add custom agent catalog
   source
3. Wait for catalog service to reload and verify via GET `/readyz`
4. Send GET to `/api/agent_catalog/v1alpha1/agents` and verify total
   count includes both sources
5. Send GET to
   `/api/model_catalog/v1alpha1/sources?assetType=agents` and verify
   both sources are listed
6. Send GET to `/api/agent_catalog/v1alpha1/agents` with
   `sourceLabel=custom-agents` to verify source filtering
7. Send GET to `/api/agent_catalog/v1alpha1/agents` with
   `sourceLabel=redhat-starter-kits` to verify default source
   filtering
8. Verify source-filtered counts match (7 custom + 15 default = 22
   total)

**Expected Results**:

- Initial count is 15 agents (default source only)
- After ConfigMap patch, total count increases to 22
- Sources endpoint lists both default and custom agent sources
- sourceLabel filtering returns disjoint sets
- Custom source agents have metadata matching the injected YAML
- Default source agents remain unchanged

**Test Data**:

```yaml
agent_catalogs:
  - name: custom-agents
    label: custom-agents
    enabled: true
    url: /opt/app-root/src/catalogs/custom-agents-catalog.yaml
```
