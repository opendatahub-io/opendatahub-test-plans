# TC-E2E-003: Filter, search, and paginate through agent catalog

**Objective**: Verify the complete user journey of filtering agents by
framework, performing keyword search, and paginating through results.

**Preconditions**:
- Both default and custom agent sources loaded (22 agents total)

**Test Steps**:
1. Send GET to `/api/agent_catalog/v1alpha1/agents/filter_options`
   to discover available filter values
2. Send GET to `/api/agent_catalog/v1alpha1/agents` with
   `filterQuery=framework = "langgraph"` to filter by framework
3. Send GET to `/api/agent_catalog/v1alpha1/agents` with
   `q=conversational` to perform keyword search
4. Send GET to `/api/agent_catalog/v1alpha1/agents` with
   `pageSize=5&orderBy=NAME&sortOrder=ASC` to paginate
5. Follow `nextPageToken` to retrieve subsequent pages until all agents
   are collected
6. Verify the total count from pagination matches the unfiltered count

**Expected Results**:
- Filter options include framework values: langgraph, crewai, autogen,
  claude-code, langflow
- Framework filter returns only agents with matching framework
- Keyword search returns agents with matching terms across fields
- Pagination returns all 22 agents across multiple pages with no
  duplicates
- Agent ordering is consistent (alphabetical by name ascending) across
  pages
