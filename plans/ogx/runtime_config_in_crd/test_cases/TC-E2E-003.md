# TC-E2E-003: Full upgrade journey

**Objective**: Verify that upgrading from RHOAI 3.4 (LLSD) to RHOAI 3.5 (OGX operator) preserves the
existing server, allows deploying a new OGXServer CR alongside it, and that all functional tests
pass on the new server.

**Preconditions**:

- OpenShift cluster with RHOAI 3.4 installed (with LLSD deployed and functional)
- Access to RHOAI 3.5 upgrade path (operator catalog)
- `oc` CLI authenticated with cluster-admin privileges
- VLLM inference endpoint accessible from the cluster
- Functional test suite available (35 tests)

**Test Steps**:

1. Verify the RHOAI 3.4 LLSD deployment is healthy:

   ```bash
   oc get pods -n redhat-ods-applications -l app=llama-stack-distribution
   ```

2. Run smoke tests against the 3.4 LLSD server to confirm baseline:

   ```bash
   # Execute baseline smoke test against the existing LLSD endpoint
   LLSD_ROUTE=$(oc get route llsd -n redhat-ods-applications -o jsonpath='{.spec.host}')
   curl -s https://$LLSD_ROUTE/health
   ```

3. Upgrade RHOAI from 3.4 to 3.5:

   ```bash
   # Trigger the operator upgrade (via subscription channel change or approval)
   oc patch subscription rhods-operator -n redhat-ods-operator --type=merge -p '
   spec:
     channel: "stable-3.5"
   '
   ```

4. Wait for the upgrade to complete:

   ```bash
   oc wait --for=condition=Available deployment/rhods-operator -n redhat-ods-operator --timeout=600s
   ```

5. Verify the old LLSD server continues running after upgrade:

   ```bash
   oc get pods -n redhat-ods-applications -l app=llama-stack-distribution
   curl -s https://$LLSD_ROUTE/health
   ```

6. Patch the DataScienceCluster (DSC) to enable the OGX component:

   ```bash
   oc patch datasciencecluster default-dsc --type=merge -p '
   spec:
     components:
       ogx:
         managementState: Managed
   '
   ```

7. Wait for the OGX operator to become ready:

   ```bash
   oc wait --for=condition=Available deployment/ogx-operator -n redhat-ods-operator --timeout=300s
   ```

8. Create a new namespace and deploy PostgreSQL for the new OGXServer:

   ```bash
   export NAMESPACE="ogx-upgrade-test"
   oc new-project $NAMESPACE

   oc create secret generic ogx-postgres-secret \
     --from-literal=password="ogx-upgrade-password" \
     -n $NAMESPACE

   # Deploy PostgreSQL (abbreviated - use standard postgres deployment)
   oc apply -f postgres-deployment.yaml -n $NAMESPACE
   oc wait --for=condition=Ready pod -l app=ogx-postgres -n $NAMESPACE --timeout=120s
   ```

9. Create secrets and apply a new OGXServer CR:

   ```bash
   oc create secret generic ogx-inference-secret \
     --from-literal=VLLM_URL="$VLLM_URL" \
     --from-literal=VLLM_API_TOKEN="$VLLM_API_TOKEN" \
     -n $NAMESPACE

   oc apply -f - <<EOF
   apiVersion: ogx.io/v1beta1
   kind: OGXServer
   metadata:
     name: ogx-upgrade-test
     namespace: $NAMESPACE
   spec:
     replicas: 1
     distribution:
       name: rh-dev
     workload:
       overrides:
         env:
           - name: VLLM_URL
             valueFrom:
               secretKeyRef:
                 key: VLLM_URL
                 name: ogx-inference-secret
           - name: VLLM_API_TOKEN
             valueFrom:
               secretKeyRef:
                 key: VLLM_API_TOKEN
                 name: ogx-inference-secret
           - name: POSTGRES_HOST
             value: "ogx-postgres.$NAMESPACE.svc.cluster.local"
           - name: POSTGRES_PORT
             value: "5432"
           - name: POSTGRES_DB
             value: "ogx_metadata"
           - name: POSTGRES_USER
             value: "ogx"
           - name: POSTGRES_PASSWORD
             valueFrom:
               secretKeyRef:
                 key: password
                 name: ogx-postgres-secret
       storage:
         size: 5Gi
   EOF
   ```

10. Wait for the new OGXServer pod to become Ready:

    ```bash
    oc wait --for=condition=Ready pod -l app=ogx-upgrade-test -n $NAMESPACE --timeout=180s
    ```

11. Run the full functional test suite against the new OGXServer
    using the deploy-and-test.sh script (which handles route creation,
    health checks, model detection, and test execution):

    ```bash
    export NAMESPACE="ogx-upgrade-test"
    export SKIP_TESTS=false
    export FUNCTIONAL_TESTS_DIR="${HOME}/code/ogx-functional-tests"
    ./deploy-and-test.sh
    ```

    Alternatively, run tests manually:

    ```bash
    OGX_ROUTE=$(oc get route ogx-upgrade-test-route \
      -n $NAMESPACE -o jsonpath='{.spec.host}')
    BASE_URL="http://${OGX_ROUTE}"
    cd "${FUNCTIONAL_TESTS_DIR}"
    BASE_URL="${BASE_URL}" \
      ./scripts/run-tests-with-providers.sh 2>&1 | tee test-results.log
    ```

12. Verify all 35 tests pass:

    ```bash
    RESULT=$(tail -1 test-results.log)
    if echo "$RESULT" | grep -q "passed"; then
      echo "PASS: functional tests passed"
    else
      echo "FAIL: $RESULT"
      exit 1
    fi
    ```

**Expected Results**:

- RHOAI 3.4 LLSD server is healthy before upgrade and smoke tests pass
- RHOAI upgrade from 3.4 to 3.5 completes without errors
- The old LLSD server continues running after upgrade (backward compatibility)
- Patching the DSC enables the OGX operator component
- A new OGXServer CR deploys successfully with its own PostgreSQL backend
- The new OGXServer health endpoint responds successfully
- All 35 functional tests pass (35/35) against the new OGXServer
