# TC-E2E-001: Declarative config — CR to running server

**Objective**: Verify the full journey from applying an OGXServer CR with providers and storage
configuration to a running, healthy server that can perform inference.

**Preconditions**:

- OpenShift cluster with OGX operator installed
- `oc` CLI authenticated with cluster-admin privileges
- A VLLM inference endpoint accessible from the cluster
- PostgreSQL container image available

**Test Steps**:

1. Create the test namespace:

   ```bash
   export NAMESPACE="e2e-runtime-config"
   oc new-project $NAMESPACE
   ```

2. Deploy PostgreSQL:

   ```bash
   oc apply -f - <<EOF
   apiVersion: apps/v1
   kind: Deployment
   metadata:
     name: ogx-postgres
     namespace: $NAMESPACE
   spec:
     replicas: 1
     selector:
       matchLabels:
         app: ogx-postgres
     template:
       metadata:
         labels:
           app: ogx-postgres
       spec:
         containers:
           - name: postgres
             image: registry.redhat.io/rhel9/postgresql-16:latest
             env:
               - name: POSTGRESQL_USER
                 value: "ogx"
               - name: POSTGRESQL_PASSWORD
                 valueFrom:
                   secretKeyRef:
                     key: password
                     name: ogx-postgres-secret
               - name: POSTGRESQL_DATABASE
                 value: "ogx_metadata"
             ports:
               - containerPort: 5432
   ---
   apiVersion: v1
   kind: Service
   metadata:
     name: ogx-postgres
     namespace: $NAMESPACE
   spec:
     selector:
       app: ogx-postgres
     ports:
       - port: 5432
         targetPort: 5432
   EOF
   ```

3. Create the secrets:

   ```bash
   export VLLM_URL="https://<inference-endpoint>/v1"
   export VLLM_API_TOKEN="<token>"

   oc create secret generic ogx-inference-secret \
     --from-literal=VLLM_URL="$VLLM_URL" \
     --from-literal=VLLM_API_TOKEN="$VLLM_API_TOKEN" \
     -n $NAMESPACE

   oc create secret generic ogx-postgres-secret \
     --from-literal=password="ogx-test-password" \
     -n $NAMESPACE
   ```

4. Wait for PostgreSQL to become ready:

   ```bash
   oc wait --for=condition=Ready pod -l app=ogx-postgres -n $NAMESPACE --timeout=120s
   ```

5. Apply the OGXServer CR:

   ```bash
   oc apply -f - <<EOF
   apiVersion: ogx.io/v1beta1
   kind: OGXServer
   metadata:
     name: ogx-e2e-test
     namespace: $NAMESPACE
   spec:
     replicas: 1
     distribution:
       name: rh-dev
     workload:
       overrides:
         env:
           - name: VLLM_URL
             valueFrom:
               secretKeyRef:
                 key: VLLM_URL
                 name: ogx-inference-secret
           - name: VLLM_API_TOKEN
             valueFrom:
               secretKeyRef:
                 key: VLLM_API_TOKEN
                 name: ogx-inference-secret
           - name: POSTGRES_HOST
             value: "ogx-postgres.e2e-runtime-config.svc.cluster.local"
           - name: POSTGRES_PORT
             value: "5432"
           - name: POSTGRES_DB
             value: "ogx_metadata"
           - name: POSTGRES_USER
             value: "ogx"
           - name: POSTGRES_PASSWORD
             valueFrom:
               secretKeyRef:
                 key: password
                 name: ogx-postgres-secret
       storage:
         size: 5Gi
   EOF
   ```

6. Wait for the operator to reconcile and the pod to become Ready:

   ```bash
   oc wait --for=condition=Ready pod -l app=ogx-e2e-test -n $NAMESPACE --timeout=180s
   ```

7. Verify a ConfigMap was created for the OGXServer:

   ```bash
   oc get configmaps -n $NAMESPACE -l app=ogx-e2e-test
   ```

8. Verify the Deployment is running with 1/1 replicas:

   ```bash
   oc get deployment ogx-e2e-test -n $NAMESPACE
   ```

9. Verify the server health endpoint responds:

   ```bash
   OGX_POD=$(oc get pods -l app=ogx-e2e-test -n $NAMESPACE -o jsonpath='{.items[0].metadata.name}')
   oc exec $OGX_POD -n $NAMESPACE -- curl -s http://localhost:8321/health
   ```

10. Verify inference works by sending a chat completion request:

    ```bash
    oc exec $OGX_POD -n $NAMESPACE -- curl -s -X POST http://localhost:8321/v1/inference/chat-completion \
      -H "Content-Type: application/json" \
      -d '{
        "model_id": "meta-llama/Llama-3.2-3B-Instruct",
        "messages": [{"role": "user", "content": "Say hello in one word."}],
        "stream": false
      }'
    ```

**Expected Results**:

- The namespace, secrets, and PostgreSQL deployment are created without errors
- After applying the OGXServer CR, the operator creates a ConfigMap containing the generated
  `config.yaml`
- A Deployment is created with 1 replica that reaches Ready state
- The ConfigMap contains provider configuration derived from `spec.providers` and storage from
  `spec.storage`
- The health endpoint at `http://localhost:8321/health` returns a successful response
- The inference endpoint returns a valid chat completion response
