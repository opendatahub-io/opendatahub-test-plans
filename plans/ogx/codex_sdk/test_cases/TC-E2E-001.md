# TC-E2E-001: Multi-step coding task — fix failing test

**Objective**: Validate that Codex CLI can complete a full coding
workflow (read failing test, navigate repo, write patch, verify fix)
through OGX with persistent session state.

**Preconditions**:

- Sample Python repo with a deliberately failing pytest test
  (e.g., assertion error in `test_calculator.py` caused by a bug
  in `calculator.py`)
- Codex CLI installed (upstream open-source, pinned version)
- `OPENAI_BASE_URL=http://llamastack-svc:8321/v1` configured
- Target OSS model serving on vLLM (port 8000) with appropriate
  `--tool-call-parser` mode
- PostgreSQL session store running on port 5432
- OGX distribution with Memories API provider registered
  in `config.yaml`

**Test Steps**:

1. Launch Codex CLI with
   `OPENAI_BASE_URL=http://llamastack-svc:8321/v1`
2. Prompt: "Run the tests, find the failing test, read the relevant
   source code, fix the bug, and verify the test passes"
3. Observe the tool-call chain produced by the model:
   `bash(pytest)` -> `read_file(test_calculator.py)` ->
   `grep_search(error pattern)` -> `read_file(calculator.py)` ->
   `edit_file(calculator.py)` -> `bash(pytest)`
4. For each tool-call turn, verify SSE streaming on
   `/v1/chat/completions` returns `delta` objects with incremental
   `tool_calls` arrays
5. After all turns complete, query PostgreSQL for the session record

**Expected Results**:

- Codex CLI completes the full tool chain with 5+ tool calls
- Each SSE stream contains properly formatted `delta.tool_calls`
  with incremental `function.name` and `function.arguments`
- Final `bash(pytest)` invocation shows all tests passing
- `session_id` is present in PostgreSQL session store
- All conversation turns (including tool results) are persisted
  in the session store

**Validation**:

- `psql -c "SELECT count(*) FROM session_turns WHERE
  session_id = '<session_id>'"` returns row count >= 5
- `psql -c "SELECT turn_type, tool_name FROM session_turns
  WHERE session_id = '<session_id>' ORDER BY created_at"`
  shows the expected tool sequence
