# TC-E2E-003: Multi-tool file exploration workflow

**Objective**: Validate that all six Codex CLI tools (bash,
read_file, write_file, edit_file, grep_search, glob_search) are
invoked and produce valid JSON arguments in a single end-to-end
workflow.

**Preconditions**:
- Sample Python project with multiple modules and directories
  (at least 5 `.py` files, some missing docstrings)
- Codex CLI configured with
  `OPENAI_BASE_URL=http://llamastack-svc:8321/v1`
- Target OSS model serving on vLLM with tool-call parser enabled
- PostgreSQL session store running

**Test Steps**:
1. Launch Codex CLI and prompt: "Find all Python files in this
   project, search for functions that don't have docstrings, read
   the first file you find with a missing docstring, add a
   docstring to the function, write a new test file for that
   function, then run the tests"
2. Observe the tool-call chain and verify all 6 tools are used:
   - `glob_search` — find `*.py` files
   - `grep_search` — search for `def ` without preceding docstring
   - `read_file` — read the identified source file
   - `edit_file` — add docstring to the function
   - `write_file` — create a new test file (e.g.,
     `test_<module>.py`)
   - `bash` — run `pytest test_<module>.py`
3. For each tool call, validate `function.arguments` is valid JSON
   conforming to the tool's parameter schema
4. Verify that SSE streaming works for each turn

**Expected Results**:
- All 6 Codex CLI tools are invoked at least once during the
  workflow
- Each `function.arguments` JSON is valid and contains the
  required fields per tool schema:
  - `bash`: `{"command": "..."}`
  - `read_file`: `{"path": "..."}`
  - `write_file`: `{"path": "...", "content": "..."}`
  - `edit_file`: `{"path": "...", ...}`
  - `grep_search`: `{"pattern": "...", ...}`
  - `glob_search`: `{"pattern": "..."}`
- Final `bash(pytest)` invocation exits with code 0
- Session state in PostgreSQL contains all tool-call turns
