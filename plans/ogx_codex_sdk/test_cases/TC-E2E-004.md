# TC-E2E-004: Concurrent Codex CLI sessions

**Objective**: Validate that multiple simultaneous Codex CLI
sessions operate independently with no cross-session context
contamination.

**Preconditions**:

- 3 separate sample repos (repo-alpha, repo-beta, repo-gamma),
  each with distinct file names and content
- 3 terminal sessions, each with a unique `session_id`
- All sessions configured with
  `OPENAI_BASE_URL=http://llamastack-svc:8321/v1`
- PostgreSQL session store running

**Test Steps**:

1. In terminal 1, launch Codex CLI targeting repo-alpha:
   "Read main.py and add error handling to the parse function"
2. In terminal 2 (simultaneously), launch Codex CLI targeting
   repo-beta: "Find all TODO comments and create a summary file"
3. In terminal 3 (simultaneously), launch Codex CLI targeting
   repo-gamma: "Write unit tests for the utils module"
4. Wait for all 3 sessions to complete their tool-call chains
5. Verify session isolation:
   - Session 1 never references files from repo-beta or repo-gamma
   - Session 2 never references files from repo-alpha or repo-gamma
   - Session 3 never references files from repo-alpha or repo-beta
6. Query PostgreSQL for all 3 session records

**Expected Results**:

- All 3 Codex CLI sessions complete their respective tasks
- No cross-session context contamination: each session's tool
  calls reference only files from its own repo
- PostgreSQL contains 3 distinct session records with separate
  `session_id` values
- Each session's turn history is complete and independent
- LlamaStack handles concurrent SSE streams without errors
  or dropped connections

**Validation**:

- `psql -c "SELECT session_id, count(*) FROM session_turns
  GROUP BY session_id"` returns 3 rows with non-zero counts
- Inspect tool-call arguments in each session's turns to confirm
  file paths belong only to the correct repo
