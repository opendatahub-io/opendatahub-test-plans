# TC-E2E-002: GPU deployment with HardwareProfile and fallback verification

**Objective**: Validate the end-to-end scenario of deploying a GPU
model with proper `nvidia-gpu-a100` HardwareProfile allocation, then
verifying that deploying without the GPU HardwareProfile prevents
silent CPU fallback (pod goes Pending).

**Preconditions**:

- OCP 4.20+ cluster with RHOAI 3.5 GA and NVIDIA GPU Operator 12.9+
- `mlserver-cuda-runtime` ClusterServingRuntime applied in
  `redhat-ods-applications` namespace
- HardwareProfile `nvidia-gpu-a100` (API group
  `infrastructure.opendatahub.io/v1`) available
- GPU nodes available in the cluster
- ResNet-50 ONNX model available in S3 storage

**Test Steps**:

1. Deploy an InferenceService with the GPU runtime and
   HardwareProfile:

   ```bash
   cat <<EOF | oc apply -f -
   apiVersion: serving.kserve.io/v1beta1
   kind: InferenceService
   metadata:
     name: resnet-gpu-e2e
     annotations:
       serving.kserve.io/deploymentMode: RawDeployment
   spec:
     predictor:
       model:
         modelFormat:
           name: onnx
           version: "1"
         runtime: mlserver-cuda-runtime
         storageUri: s3://models/resnet-50-onnx/
   EOF
   ```

2. Wait for the InferenceService to reach `Ready`:

   ```bash
   oc wait inferenceservice/resnet-gpu-e2e --for=condition=Ready \
     --timeout=300s
   ```

3. Verify GPU inference works:

   ```bash
   URL=$(oc get inferenceservice resnet-gpu-e2e \
     -o jsonpath='{.status.url}')
   curl -s -o /dev/null -w "%{http_code}" \
     -X POST "${URL}/v2/models/resnet-gpu-e2e/infer" \
     -H "Content-Type: application/json" \
     -d @resnet-input.json
   ```

4. Delete the GPU InferenceService:

   ```bash
   oc delete inferenceservice resnet-gpu-e2e
   ```

5. Deploy a new InferenceService using the GPU runtime WITHOUT a GPU
   HardwareProfile (no `nvidia-gpu-a100` association):

   ```bash
   cat <<EOF | oc apply -f -
   apiVersion: serving.kserve.io/v1beta1
   kind: InferenceService
   metadata:
     name: resnet-gpu-nohw
     annotations:
       serving.kserve.io/deploymentMode: RawDeployment
   spec:
     predictor:
       model:
         modelFormat:
           name: onnx
           version: "1"
         runtime: mlserver-cuda-runtime
         storageUri: s3://models/resnet-50-onnx/
   EOF
   ```

6. Wait 60 seconds and verify the pod is Pending:

   ```bash
   oc get pod \
     -l serving.kserve.io/inferenceservice=resnet-gpu-nohw \
     -o jsonpath='{.items[0].status.phase}'
   ```

7. Verify no inference is accessible:

   ```bash
   oc get inferenceservice resnet-gpu-nohw \
     -o jsonpath='{.status.conditions[?(@.type=="Ready")].status}'
   ```

**Expected Results**:

- Step 2: InferenceService reaches `Ready` with GPU HardwareProfile
- Step 3: Inference returns HTTP `200` with correct prediction
- Step 6: Pod status is `Pending` without GPU HardwareProfile
- Step 7: InferenceService `Ready` condition is `False`
- No silent CPU fallback occurs — the model does not serve inference
  without proper GPU allocation
