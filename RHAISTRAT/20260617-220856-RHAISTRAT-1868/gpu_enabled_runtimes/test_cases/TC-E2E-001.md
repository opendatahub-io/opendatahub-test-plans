# TC-E2E-001: Data scientist deploys GPU model via Dashboard and runs inference

**Objective**: Validate the complete user journey of a data scientist
deploying a GPU-accelerated ONNX model through the RHOAI Dashboard
using the `mlserver-cuda-runtime` and successfully running inference
against it.

**Preconditions**:

- OCP 4.20+ cluster with RHOAI 3.5 GA and NVIDIA GPU Operator 12.9+
- `mlserver-cuda-runtime` ClusterServingRuntime applied in
  `redhat-ods-applications` namespace with Dashboard label and
  recommended-accelerators annotation
- HardwareProfile `nvidia-gpu-a100` (API group
  `infrastructure.opendatahub.io/v1`) available
- GPU nodes available in the cluster
- ResNet-50 ONNX model stored in S3-compatible storage
- `data-scientist` user with namespace access
- RHOAI Dashboard accessible

**Test Steps**:

1. Log in to the RHOAI Dashboard as `data-scientist`.
2. Navigate to Data Science Projects and select the test project.
3. Initiate a model deployment (Deploy model).
4. Select the `mlserver-cuda-runtime` runtime from the runtime
   dropdown.
5. Verify the GPU HardwareProfile (`nvidia-gpu-a100`) is shown as a
   recommended accelerator option and select it.
6. Configure the model storage URI pointing to the ResNet-50 ONNX
   model.
7. Submit the deployment.
8. Wait for the model deployment to show "Ready" status in the
   Dashboard.
9. Obtain the inference endpoint URL from the Dashboard or via CLI:

   ```bash
   URL=$(oc get inferenceservice resnet-gpu \
     -o jsonpath='{.status.url}')
   ```

10. Verify the model readiness endpoint:

    ```bash
    curl -s -o /dev/null -w "%{http_code}" \
      "${URL}/v2/models/resnet-gpu/ready"
    ```

11. Send an inference request:

    ```bash
    curl -s -X POST "${URL}/v2/models/resnet-gpu/infer" \
      -H "Content-Type: application/json" \
      -d @resnet-input.json
    ```

12. Verify the inference response contains a correct prediction.

**Expected Results**:

- GPU runtime (`mlserver-cuda-runtime`) is visible and selectable in
  the Dashboard runtime dropdown
- GPU HardwareProfile (`nvidia-gpu-a100`) appears as recommended when
  GPU runtime is selected
- Model deployment reaches "Ready" status in the Dashboard
- Model readiness endpoint returns HTTP `200`
- Inference request returns HTTP `200` with a valid prediction
  output matching the expected class for the reference input
- Pod is running on a GPU node with `nvidia.com/gpu` resource
  allocated
